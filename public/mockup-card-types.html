Design rules for all card templates
  1. Art and text live in separate zones, never overlapping. If a template has hero art, give it a fixed-height stage (e.g. .art-zone) and put text only in the .text-zone below. Now that SVGs are transparent, an absolutely-positioned graphic crashes into text and becomes unreadable.
  2. If art must be behind text, opacity caps at 15%. Stat card uses 8%, Quote card uses ~15% via multiply blend. Above 20% on a content area = unreadable headline.
  3. Recolor SVG strokes to match the card background when the asset is meant to be hero. Cover card uses filter: brightness(0) invert(1) to make the navy cityscape strokes white-on-teal. Tip card relies on the corner art being faded enough that color doesn’t matter.
  4. Wordmark rule is honoured per card automatically — “Nana” stays white on dark backgrounds, navy on light. The template owns this; you don't pick.
  5. Text never sits flush against an edge. Minimum 28-32px padding all sides; chips/labels sit 4-8px in from card padding line.
Things to consider before picking which to build first
  • Templates (Option 1) are fast to make once they exist. Each new card = pick template + pick icon + type 2-4 fields = 30 seconds. Five templates above = five distinct content shapes you can mix daily.
  • Claude-generated (Option 2) takes ~30s-2min per post, but each one looks different. Better for “hero” posts you don't make often. Worse for daily drip.
  • You can do both. Templates handle daily Glossary / Tip / Stat content. Claude-generated handles the occasional “featured insight” post. They share the same approve / Thai / download pipeline.
